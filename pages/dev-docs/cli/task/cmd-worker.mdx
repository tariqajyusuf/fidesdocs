# Command: worker

> **Hands-on:** To understand workers in Fides try the [Product Deployment](/dev-docs/configuration/deployment) tutorials.

The ```fides worker``` starts a celery worker listening to Fides server.
This is extremely helpful for:

1. Scaling the concurrent privacy requests that Fides can process.
2. Avoiding asynchronous race conditions and performance issues.

## Usage
Usage: ```fides workers```

Running this command will start a [Celery Worker](https://docs.celeryq.dev/en/stable/userguide/workers.html#starting-the-worker). As you can see in the example below, the worker is configured to listen for privacy request tasks and execute privacy requests. For more in-depth information on privacy requests read the [Fides Privacy Requests](/tutorials/privacy-requests-get-started) guide.
