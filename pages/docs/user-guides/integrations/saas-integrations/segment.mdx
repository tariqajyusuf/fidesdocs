# Segment
[Segment](https://segment.com/) is a customer data platform (CDP) that helps businesses collect and manage customer data.

## Prerequisites

In order to [integrate](/docs/user-guides/integrations) with Segment, you'll need to collect the following information:

| Name                       | Description                                                                                                                             |
| :------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------- |
| **Domain***                | Your Segment API URL (default: `platform.segmentapis.com`)  |
| **Profiles domain***       | Your Segment profiles API URL (default: profiles.segment.com) |
| **Workspace***             | Your Segment workspace ID. See Segment's guide on [workspaces](https://segment.com/docs/segment-app/workspace-home/). |
| **Access token***          | Your Segment access token. Follow Segment's [guide for creating access tokens](https://segment.com/docs/api/config-api/authentication/#create-an-access-token).|
| **Space ID***          | Your Segment space ID.  See [Segment's guide on spaces.](https://segment.com/docs/unify/identity-resolution/space-setup/). |
| **Access secret***         | Your access secret for your profiles domain. Follow Segment's docs [to retrieve this info.](https://segment.com/docs/unify/profile-api/#authentication). |

## Integrating with Segment
To integrate an existing system with Segment:

1. Navigate to **Data map** â†’ **View Systems** and choose the system that you want to connect.
2. Click on the **Integrations** tab.
3. Pick the vendor or system type that you want to integrate with from the **Connection type** drop-down menu.
4. Complete the required fields for the integration and click **Save**.

You may confirm your connection is working using the **Test connection** button, if desired.

To learn more, please see our guide for [Managing Integrations](/docs/user-guides/integrations/manage-integrations).

## Technical detail

Fides utilizes _API endpoints_ to access the _API service_ for a SaaS application. An _API service_ is the programmatic interface through which Fides can query and update data within an application to access, rectify, or delete personal information. Each SaaS tool will have a unique set of endpoints to enable privacy functions.

### Segment endpoints
In the table below, you can find which endpoints are used for each type of privacy function. Clicking on the endpoint will take you to the documentation for that endpoint.

| Privacy function | Endpoint | Description |
|----|----|----|
| **Access Requests** | [Segment user](https://segment.com/docs/unify/profile-api/#get-a-profiles-metadata) | Used to retrieve profile metadata. |
| **Access Requests** | [Track events](https://segment.com/docs/unify/profile-api/#get-a-profiles-events) | Used to retrieve a profile's historical events |
| **Access Requests** | [Traits](https://segment.com/docs/unify/profile-api/#get-a-profiles-traits) |  Used to retrieve a profile's traits |
| **Access Requests** | [External IDs](https://segment.com/docs/unify/profile-api/#get-a-profiles-external-ids) |  Used to retrieve a profile's external IDs |
| **Delete Requests** | [Data protection request](https://segment.com/docs/privacy/user-deletion-and-suppression/) | Used to suppress a user or delete all their information |
