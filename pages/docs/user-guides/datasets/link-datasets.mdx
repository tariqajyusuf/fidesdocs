# Linking datasets 

In this section, we will explore the process for linking [datasets](/user-guides/datasets) to [integrations](/user-guides/integrations) so that privacy requests can be executed against all of your data, whether in a database or third party SaaS application.


## Linking a dataset to an integration
Database integrations require a dataset to describe the table or collection schemas and relationships. To link a dataset to an integration, you can either [create a new dataset](#creating-a-new-dataset) or [link an existing dataset](#linking-an-existing-dataset).

### Creating a new dataset 
To create a new dataset while configuring a [database integration](/user-guides/integrations#2-collect-the-information-that-you-will-need-to-connect-to-the-system), you may use the _dataset editor_ by:
1. Clicking on the editor icon.
2. Pasting or typing in the YAML configuration.

<Screenshot img="/assets/img/integrations/create-datasets.gif" description="Create a dataset" /> 

>You may edit an existing dataset at any time by returning to the database integation and selecting the dataset editor icon.

To learn additional ways to create datasets, please see our guide for [Creating datasets](/user-guides/datasets/generate-datasets).


### Linking an existing dataset 
To link an existing dataset while configuring a [database integration](/user-guides/integrations/manage-integrations#2-configure-the-connection-information-and-test-the-integration),
select the appropriate dataset from the drop-down menu.
