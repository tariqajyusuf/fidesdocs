# Fides 2.18 Release notes
August 7, 2023

## Fides for Business 2.18.0

- **Added reporting for served notices**: Fides now captures which notices are served along with user geography, device ID, etc.


## Fides OSS 2.18.0

- **UI support for Auth0 Authorization flow**: Integrations which use Auth0 for authentication have an improved flow for authorization. See an example flow in the [Salesforce documentation](https://ethyca.com/docs/user-guides/integrations/saas-integrations/salesforce).
- **Simplified Access packages**: the Access package root directory now contains a more user-friendly `welcome.html` and all of the data necessary to display the report has been aggregated under the `data` directory.
- **Fixes**
    - System and integration identifiers are now auto-generated #3868
    - Removed manual system selection screen #3865
    - Fixed datamap zoom for low system counts [#3835](https://github.com/ethyca/fides/pull/3835)
    - Fixed connector forms with external dataset reference fields [#3873](https://github.com/ethyca/fides/pull/3873)
    - Changed max width of form components in "system information" form tab [#3864](https://github.com/ethyca/fides/pull/3864)
    - Simplified the database health check to improve /health performance [#3884](https://github.com/ethyca/fides/pull/3884)
    - Changes in the data directory now trigger a server reload (for local development) [#3874](https://github.com/ethyca/fides/pull/3874)
- **Integrations**:
    - **Recurly**: we now offer a Recurly integration for Access requests [#3595](https://github.com/ethyca/fides/pull/3595)
