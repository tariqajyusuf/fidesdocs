# Doordash
[Doordash](https://www.doordash.com/) is a food delivery service that connects customers with local restaurants.

## Prerequisites

<Callout emoji="" type="info">An owned database must be configured to store Doordash data for privacy request processing. To learn how to integrate with a database, please see our guide for [integrating with databases]/docs/user-guides/integrations/databases). </Callout>


In order to [integrate](/docs/user-guides/integrations) with Doordash, you'll need to collect the following information:

| Name                       | Description                                                                                                                             |
| :------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------- |
| **Domain**                 | Your Doordash API domain (default: openapi.doordash.com) |
| **Developer ID***          | Your Doordash developer ID. Follow Doordash docs to [create an access key](https://developer.doordash.com/en-US/docs/drive_classic/how_to/manage_credentials/#create-an-access-key) and retrieve the developer id |
| **Key ID***                | Your Doordash access key ID. Follow Doordash docs to [create an access key](https://developer.doordash.com/en-US/docs/drive_classic/how_to/manage_credentials/#create-an-access-key) and retrieve the key id |
| **Secret***        | Your Doordash signing secret. Follow Doordash docs to [create an access key](https://developer.doordash.com/en-US/docs/drive_classic/how_to/manage_credentials/#create-an-access-key) and retrieve the signing secret|
| **Delivery ID field***  | The location in your owned database where you store Doordash delivery IDs. Format: `dataset.collection.field` |

## Integrating with Doordash
To integrate an existing system with Doordash:

1. Navigate to **Data map** â†’ **View Systems** and choose the system that you want to connect.
2. Click on the **Integrations** tab.
3. Pick the vendor or system type that you want to integrate with from the **Connection type** drop-down menu.
4. Complete the required fields for the integration and click **Save**. 

You may confirm your connection is working using the **Test connection** button, if desired.

To learn more, please see our guide for [Managing Integrations](/docs/user-guides/integrations/manage-integrations).

## Technical detail 

Fides utilizes _API endpoints_ to access the _API service_ for a SaaS application. An _API service_ is the programmatic interface through which Fides can query and update data within an application to access, rectify, or delete personal information. Each SaaS tool will have a unique set of endpoints to enable privacy functions. 

### Doordash Endpoints
In the table below, you can find which endpoints are used for each type of privacy function. Clicking on the endpoint will take you to the documentation for that endpoint.

| Privacy function | Endpoint | Description |
|----|----|----|
| **Access Requests** | [Delivery](https://developer.doordash.com/en-US/api/drive/#tag/Delivery/operation/GetDelivery) | Get information about a user's delivery |
