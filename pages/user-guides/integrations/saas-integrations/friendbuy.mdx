# Friendbuy

[Friendbuy](https://www.friendbuy.com) is a customer referral platform that helps businesses create and manage referral programs to acquire new customers through word-of-mouth marketing.

## Prerequisites

<Callout emoji="" type="info">An owned database must be configured to store Friendbuy data for privacy request processing. To learn how to integrate with a database, please see our guide for [integrating with databases](../databases). </Callout>

In order to [integrate](../) with Friendbuy, you'll need to collect the following information:

| Name                    | Description                                                                                                                         |
| :---------------------- | :---------------------------------------------------------------------------------------------------------------------------------- |
| **Domain\***            | The Friendbuy API domain (default: api.friendbuy.com/v2)                                                                                   |
| **API token\***             | Your long-lived API token. Follow Friendbuy's docs to [obtain your long-lived token](https://developers.friendbuy.com/legacy/#security) |
| **Friendbuy User ID\*** | The location in your owned database where you store Friendbuy User Ids, in the following format: `dataset.collection.field`                          |

## Technical detail

Fides utilizes _API endpoints_ to access the _API service_ for a SaaS application. An _API service_ is the programmatic interface through which Fides can query and update data within an application to access, rectify, or delete personal information. Each SaaS tool will have a unique set of endpoints to enable privacy functions.

### Friendbuy Endpoints

In the table below, you can find which endpoints are used for each type of privacy function. Clicking on the endpoint will take you to the documentation for that endpoint.

| Privacy function              | Endpoint                                                                           | Description                               |
| ----------------------------- | ---------------------------------------------------------------------------------- | ----------------------------------------- |
| **Access Requests**           | [Customers](https://developers.friendbuy.com/legacy/#get-customers-id)             | Returns information about a customer.     |
| **Update or Delete Requests** | [Delete User Data](https://developers.friendbuy.com/legacy/#post-user_data-delete) | Used to mask or erase records for a user. |
